# Co-Write

**Co-Write** — це високопродуктивний вебдодаток для спільного редагування коду у реальному часі без виникнення конфліктів. 

В основі проєкту лежить технологія **CRDT (Conflict-free Replicated Data Type)** за допомогою бібліотеки **Automerge**, яка забезпечує безшовне об'єднання змін від різних користувачів, а комунікація між клієнтом та сервером відбувається миттєво через **WebSockets**.

---

## Основні можливості

- **Редагування у реальному часі:** Спільна робота над кодом з миттєвою синхронізацією.
- **Вирішення конфліктів без болю:** Використання Automerge гарантує, що всі копії документа зрештою прийдуть до єдиного стану без блокування інтерфейсу.
- **Підсвічування синтаксису:** Потужний вбудований редактор коду **CodeMirror 6** (з підтримкою синтаксису Rust, One Dark темою тощо).
- **Документація API (Swagger):** Інтегрована інтерактивна документація Swagger UI для розробників.
- **Повна контейнеризація:** Простий запуск усього стеку за допомогою Docker та Docker Compose.

---

## Технологічний стек

### Backend (Серверна частина)
- **Мова:** [Rust](https://www.rust-lang.org/) — гарантує безпеку пам'яті, високу швидкість та надійність.
- **Вебфреймворк:** [Actix-web](https://actix.rs/) та `Actix-ws` для обслуговування WebSockets.
- **База даних:** [PostgreSQL](https://www.postgresql.org/) (асинхронна робота через [SQLx](https://github.com/launchbadge/sqlx)).
- **Кеш та Pub/Sub:** [Redis](https://redis.io/) для тимчасового збереження кімнат та брокера повідомлень.
- **Документація:** [Utoipa](https://github.com/juhakg/utoipa) для генерації OpenAPI специфікації та Swagger UI.

### Frontend (Клієнтська частина)
- **Фреймворк:** React 19 + Vite (швидка збірка).
- **Стейт-менеджер:** Redux Toolkit + React Redux.
- **Редактор:** CodeMirror 6 (тема One Dark, підсвічування Rust).
- **Синхронізація:** `@automerge/automerge` (робота з бінарними sync-повідомленнями CRDT).

## Налаштування та запуск локально

Ви можете запустити проєкт двома способами: найпростішим через **Docker Compose**, або запустивши кожен компонент **вручну** для активної розробки.

### Передумови
Переконайтеся, що на вашому комп'ютері встановлено:
- **Docker** та **Docker Compose** (для першого способу)
- **Node.js** (v18+) & **npm** (для ручного запуску клієнта)
- **Rust (rustc & cargo v1.80+)** (для ручного запуску сервера)

Найпростіший спосіб запустити весь стек (Бекенд + Фронтенд + БД + Redis) однією командою:

1. Перевірте наявність файлу `.env` в корені проєкту. У ньому вже містяться базові налаштування розробки.
2. Запустіть контейнери:
   ```bash
   docker compose up --build -d
   ```
3. Після успішного запуску додаток буде доступний за такими адресами:
   - **Frontend (Клієнт):** [http://localhost:3000](http://localhost:3000)
   - **Backend (Сервер API):** [http://localhost:8080](http://localhost:8080)
   - **Swagger UI (Документація API):** [http://localhost:8080/swagger-ui/](http://localhost:8080/swagger-ui/)

*Щоб зупинити контейнери, виконайте:*
```bash
docker compose down
```

## Як працює синхронізація (Automerge CRDT)

1. **Ініціалізація:** При створенні чи відкритті документа клієнт встановлює постійне з'єднання з сервером через WebSocket.
2. **Внесення змін:** Коли користувач друкує текст у редакторі CodeMirror, фронтенд фіксує ці зміни та передає їх у локальний документ **Automerge**.
3. **Обмін повідомленнями:** Automerge створює бінарне повідомлення синхронізації (sync message). Це повідомлення відправляється на сервер через WebSocket.
4. **Обробка на сервері:** Бекенд на Rust отримує бінарний пакет, застосовує його до своєї копії документа Automerge, зберігає зміни в базі даних та за допомогою Redis Pub/Sub транслює ці оновлення всім іншим клієнтам, підключеним до цієї ж кімнати.
5. **Конфлікт-фри злиття:** Інші клієнти застосовують отримане повідомлення до своєї копії Automerge. Алгоритми CRDT гарантують, що конфлікти вирішуються автоматично (наприклад, одночасне введення тексту в одне й те саме місце не призведе до перезапису, а об'єднає обидва тексти).
